# Marknote 설계 컨텍스트

## UI 디자인 철학: "미니멀, 그러나 여유있게"

### 핵심 원칙
- **미니멀**: 장식 최소화. 불필요한 테두리, 그림자, 그라디언트 없음
- **여유있는 패딩**: 요소 간 간격은 넉넉하게. 답답하지 않은 레이아웃
- **미니멀 인디케이터**: 선택/활성 상태는 짧은 라인(언더라인 또는 좌측 라인)으로 표현
- **슬라이딩 하이라이트**: 호버 시 배경이 부드럽게 따라다니는 애니메이션
- **플랫하지만 고급스럽게**: 볼록/오목 입체감 대신 색상 대비와 미세한 라인으로 깊이 표현

### 구체적 규칙
- **활성 상태**: 엑센트 컬러(#1a73e8) 짧은 라인 (14px 폭, 2px 높이)
- **호버**: 연한 회색(#f5f5f5) 슬라이딩 배경, 0.2s cubic-bezier(0.4, 0, 0.2, 1)
- **버튼 간 간격**: gap 0. 하이라이트가 빈틈 없이 이동
- **행 높이**: 34px 통일 (툴바 버튼, 사이드바 파일, 폴더)
- **텍스트**: 본문 14px, 라벨 11-12px, 색상 계층 (#222 > #333 > #555 > #888 > #999)
- **전환 속도**: 0.1-0.2s. 빠르고 부드럽게
- **엑센트**: Google Blue (#1a73e8) 단일 색상
- **라운딩**: 6-8px (버튼/카드), 12px (모달)

### 데이터 철학
- 모든 문서는 표준 마크다운(.md) 파일로 로컬에 저장
- DB 테이블은 로컬 SQLite, CSV/마크다운 export 가능
- 서비스 없이도 데이터는 온전히 사용자 소유
