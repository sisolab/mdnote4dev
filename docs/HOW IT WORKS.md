# Marknote — How It Works

> 사람용. 설계 철학과 동작 원리.
> 코드 상세는 → [REFERENCE.md](./REFERENCE.md)

## 앱 철학

- **"파일 자체에 데이터"** — 모든 문서는 표준 마크다운(.md), 메타데이터는 frontmatter
- 서비스 없이도 데이터는 온전히 사용자 소유
- 다른 마크다운 앱에서도 문서를 열 수 있어야 함
- 이미지/첨부 저장 시 상대경로 `./.assets/` 사용 (절대경로 변환 금지)

## 아키텍처

- **Tauri v2** + React 19 + TypeScript + Vite + Tailwind CSS v4
- **TipTap** (ProseMirror) WYSIWYG 에디터 + @tiptap/markdown
- **Zustand** 상태 관리 (persist 미들웨어로 localStorage 저장)
- **Google Fonts CDN** 동적 폰트 로드 (오프라인 시 시스템 fallback)

## 마크다운 변환 — 왜 이렇게 하나

@tiptap/markdown 공식 확장으로 마크다운↔에디터 변환. 이전에는 직접 만든 regex 파서였지만 엣지 케이스가 끝없어서 교체.

**이미지**: 표준 마크다운 `![](url)`은 width/align 속성을 담을 수 없어서 `<img>` HTML 태그로 저장. 에디터 내부에서는 Tauri의 asset URL(`http://asset.localhost/...`)을 쓰지만, 파일에는 반드시 상대경로(`./.assets/...`)로 변환해서 저장.

**첨부파일**: `[filename](./.assets/filename)` 표준 링크로 저장. 로드 시 @tiptap/markdown이 이걸 link mark 또는 plain text로 파싱하므로, 텍스트 패턴 매칭으로 fileAttachment 노드로 변환.

## 에셋 라이프사이클 — 삭제와 되돌리기

이미지/첨부파일을 삭제하면 즉시 지우지 않고 `.trash/` 폴더로 이동. Ctrl+Z로 되돌리면 `.trash/`에서 `.assets/`로 복원. 앱 종료 시 `.trash/` 내용을 OS 휴지통으로 최종 이동.

문서 자체를 삭제할 때는 `.assets/` 폴더에서 문서명 접두사로 매칭하는 파일들을 찾아 `.trash/`로 이동. 마크다운 content를 파싱하지 않으므로 저장 여부와 무관하게 동작.

## 탭 시스템

3종의 특수 탭(검색/첨부/탭 탐색기)은 항상 탭바 왼쪽에 고정. 문서 탭은 드래그로 순서 변경 가능.

**고정(pin)**: 탭 우클릭 → 고정. 핀 아이콘 표시, Ctrl+W 보호, 고정탭 바로 뒤로 정렬.

**탭 전환 시 편집 보존**: 에디터가 포커스를 잃을 때(blur) 마크다운을 탭 content에 동기화. 탭을 전환해도 편집 내용이 사라지지 않음.

**isDirty 닫기**: 미저장 탭을 닫으면 확인 다이얼로그(저장 후 닫기/저장 안 함/취소). `closeTab`은 `requestAnimationFrame`으로 호출 (React 렌더링 충돌 방지).

## 설정 시스템

오른쪽 사이드 패널(400px)에 4개 탭:
- **설정**: 테마(4종), 강조색(7색), 저장 모드(5종)
- **스타일**: 타이포그래피/레이아웃/코드블록/줄간격 슬라이더 + 프리셋
- **디자인**: H1-H3/인용문/코드블록/수평선 스타일 선택 + 라이브 미리보기
- **글꼴**: 언어별 본문 폰트 + 코드 폰트, 클릭 즉시 적용

## 사이드바 — 즐겨찾기 폴더 방식

기존 파일 시스템의 아무 폴더나 즐겨찾기로 등록. OneDrive 등 클라우드 동기화 폴더를 그대로 사용 가능. FS 감시로 외부 변경을 실시간 반영.

파일 트리에서 .md 파일이 상위 정렬, 비마크다운 파일은 옅게 표시. 모든 파일 작업(이동/삭제/이름변경)은 Ctrl+Z로 되돌리기 가능.

## 창 크기 관리

동적 최소 크기: `사이드바 폭 + 에디터 고정폭 + 여백`. 사이드바 토글 시 250ms 애니메이션으로 창 크기 조절.
